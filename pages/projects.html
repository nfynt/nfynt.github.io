Future Ride (C#/Unity/AI)

Future Ride, among top 20 game at Pocket Gamer Connect, Bangalore 2016. It's an arcade style local multiplayer game that supports 1v1, 2v2, and single player mode. It's gameplay is inpired from rocket league and we tried to bring the same experience over mobile and local PC. The player can use their xbox or ps controller to connect with system and enjoy multiplayer mode. In single player mode the user plays against basic reactive AI whose only purpose is to save and score any how.
Future Ride is developed using Unity and C#. I have worked upon the project from ideation to development stage along with Abhinav who is the co-founder of AIADP studio.

  • Gameplay: Local multiplayer game with option of single player against reactive AI. Wrote the whole gameplay and mechanics of the game that include scoring, power ups and vehicle controls (i.e. driving and stunts). Architected and implemented the whole bot opponent logic. Used Unity 5’s uGUI to create flexible views.
  • Engine: Implemented AI freeplay behaviors, game states and goals. Implemented controller inputs and key bindings. Used Unity 5's new UGUI for application interface.



FABRIK (C#/Unity/IK/Heatmaps/CG)

FABRIK, is an heuristic iterative algorithm of solving IK. Here I have tried to implement this novel algorithm for solving joint transform for tracking of target object in XZ-plane and identifying each joints configuration in the process. Further with reference from published paper we will try to implement few tests to validate the results. Firstly I have created articulated chain system with 5 links having link length of 5, 4, 3, 2, 1 respectively. And the complete movement is restricted on XZ-plane with allowed rotation of each joint along Y-axis only. The XZ-plane is represented with grid lines which is used to plot/define target position which triggers FABRIK algo to calculate joint configuration with respect to defined temporal and permanent constraints of angular velocity, swing, and twist.

  • Controls: Orbit camera movement with right click to move around and mouse scroll wheel to zoom in and out of grid. Left clicking anywhere on the grid sets new target position.
  • Constraints: This project implements one temporal constraint of angular velocity and two permanent constraint of swing and twist value for each joint.
  • Heatmaps: The colour in heatmap varies between shades of “RED” and “GREEN”, where red is for not-reachable positions as opposed to green which represents reachable positions. Also, smaller the size of circle means greater is the accuracy of end-effector to navigate to the point. The visual representation is managed from surface shader applied on the grid plane.
  • Shader: Generally CG doesn't allow array data types to be passed from unity as parameter and hence custom property is created to receive float4 array in fragment shader.



Once Upon A Bike

A new construction genre game inspired from Besiege and Poly Bridge where the objective of each level is to carry pre-defined users across the level by constructing your own ride which can sustain all the level difficulties. The users vary from a boy to adult and even to animals, each of them different weights and strength to hold on to bike. The difficulties in level varies from railway track to cattle on road and sometimes para-gliding from hill.

  • Programmer for gameplay and mechanics. Took the project from POC and worked on new components and parts of the vehicle.
  • Programmed GIF encoder for the gameplay which can be saved, replayed, edited and shared after the level is completed.
  • Game leaderboard. each players high score gets saved on cloud in serialised JSON container which is then used to populate and present Histograms and leaderboards.
  • Procedural environments and custom level editing options, where user can draw their own course on 2D grid which generates up the 3D level and saves it locally.



Distance Measurement(C#/Unity/Spatial Mapping/Hololens)

HoloDistance, was a POC project where the idea was to test the accuracy and feasibility of distance measurement using Hololens. Nuspace (C#/C++/Unity/etc)

Nuspace, is an umbrella product which encompasses 5 other individual product, each specific and targeted for different use case. I have worked on 2 products - Nudesign and Nusupport. Nudesign is virtual 3D conferencing with all collaboration options such as voice, avatar, 3D environments, etc. But on top of all this we have option for dynamic contents and manipulation tools such as grid system, cross sectional of object, Simulation and animation, etc.
Nusupport is more focused for on demand remote assistance and remote monitoring of operations. In this user can share their live camera stream, screen sharing, embedded browser, etc.

  • NuDesign: recording and playing back of entire session, the user can record the entire 3D online session and save it locally which can then be saved to review or reference purpose afterwards.
  • NuDesign: interactive object update and design tools feature set, that allows the user to interact with dynamically loaded model.
  • NuDesign: simulation of evacuation scenario in industry environment and fluid motion simulation.
  • Nusupport: architecting and implementing the entire solution. Extending support for smart wearable devices.
  • Nusupport: Live streaming of user stream and broadcasting it to all the other users in current session only. The current live stream can also be recorded by "Host" and saved/shared in mp4 format.
